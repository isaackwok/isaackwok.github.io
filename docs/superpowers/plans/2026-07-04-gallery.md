# Gallery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the `/gallery` placeholder with a curated single-column photo portfolio driven by one YAML data file and Astro's build-time image pipeline.

**Architecture:** A `gallery` content collection loads `src/data/gallery.yaml` via Astro's `file()` loader with a custom YAML parser that derives each entry's `id` from its image filename and its `order` from array position. `src/pages/gallery.astro` renders the sorted collection as a sequence of `<figure>` elements using `<Image />` (Sharp, build-time WebP). No client-side JavaScript.

**Tech Stack:** Astro 7, Tailwind CSS v4, `yaml` (new dependency), `sharp` (new devDependency, used by Astro's image service and by a throwaway sample-photo script).

**Spec:** `docs/superpowers/specs/2026-07-04-gallery-design.md`

- [ ] **Step 2: Create the content collection config**

Create `src/content.config.ts` (this repo has no content collections yet — this is a new file, and Astro picks it up by convention at `src/content.config.ts`):

```typescript
import { defineCollection } from "astro:content";
import { file } from "astro/loaders";
import { z } from "astro/zod";
import { parse } from "yaml";

const gallery = defineCollection({
    loader: file("src/data/gallery.yaml", {
        parser: (text) =>
            (parse(text) ?? []).map(
                (entry: Record<string, string>, index: number) => ({
                    ...entry,
                    id: entry.image.split("/").pop()!.replace(/\.[^.]+$/, ""),
                    order: index,
                }),
            ),
    }),
    schema: ({ image }) =>
        z.object({
            image: image(),
            alt: z.string().min(1),
            caption: z.string().min(1),
            order: z.number(),
        }),
});

export const collections = { gallery };
```

The parser derives `id` from the image basename and injects `order` from array position, so reordering the gallery is just moving lines in the YAML — no numbers to maintain.

- [ ] **Step 1: Rewrite the gallery page**

Replace the entire contents of `src/pages/gallery.astro` with:

```astro
---
import { getCollection } from "astro:content";
import { Image } from "astro:assets";
import Layout from "../layouts/Layout.astro";
import Nav from "../components/Nav.astro";

const photos = (await getCollection("gallery")).sort(
    (a, b) => a.data.order - b.data.order,
);
---

<Layout
    title="Gallery — Isaac Kwok"
    description="Photo gallery of Isaac Kwok — photographs and moments."
>
    <main class="mx-auto max-w-xl px-6 pt-10 pb-24 sm:px-8 sm:pt-14 sm:pb-32">
        <Nav />

        <header class="relative mt-16 sm:mt-28">
            <span
                class="tategaki absolute -left-14 top-1 hidden select-none font-display text-sm text-hai/70 lg:block"
                aria-hidden="true">攝影</span
            >
            <a
                href="/"
                class="text-xs font-medium tracking-[0.35em] text-hai uppercase transition-colors hover:text-sumi"
                >Isaac Kwok</a
            >
            <h1
                class="mt-6 font-display text-4xl font-medium tracking-tight sm:text-5xl"
            >
                Gallery
            </h1>
        </header>

        <div class="mt-20 sm:mt-24">
            {
                photos.map((photo, index) => {
                    const isPortrait =
                        photo.data.image.height > photo.data.image.width;
                    return (
                        <figure
                            class:list={[
                                "mt-16 first:mt-0 sm:mt-20",
                                isPortrait && "w-[70%]",
                            ]}
                        >
                            <Image
                                src={photo.data.image}
                                alt={photo.data.alt}
                                widths={isPortrait ? [384, 768] : [512, 1024]}
                                sizes={
                                    isPortrait
                                        ? "(min-width: 640px) 358px, 70vw"
                                        : "(min-width: 640px) 512px, calc(100vw - 48px)"
                                }
                                format="webp"
                                quality={80}
                                loading={index === 0 ? "eager" : "lazy"}
                                fetchpriority={index === 0 ? "high" : "auto"}
                                class="w-full"
                            />
                            <figcaption class="mt-3 text-sm text-hai">
                                {photo.data.caption}
                            </figcaption>
                        </figure>
                    );
                })
            }
        </div>

        <footer
            class="mt-32 flex items-baseline justify-between text-sm text-hai sm:mt-40"
        >
            <p>&copy; 2026 Isaac Kwok</p>
        </footer>
    </main>
</Layout>
```

Layout notes (from the spec): same `max-w-xl` container and header skeleton as other pages; landscape photos fill the container, portrait photos get `w-[70%]` left-aligned; orientation is read from the image's intrinsic metadata (`image.height > image.width`) — never a manual flag; the first photo loads eagerly with high fetch priority, the rest lazy.
