# [Nuxt] 讓 Vuex 與 LocalStorage 同步

# 前言

在使用 Vuex 時，總有些例如 user preference 或 data 想儲存到 Client-Side Storage，本篇介紹如何用 Nuxt + Vuex-persist 套件達成 Vuex 與 LocalStorage 同步。

# Step 1: 安裝 Vuex-Persist

yarn add vuex-persist
# or
npm install vuex-persist

# Step 2: 建立 plugin

在 Nuxt 裡，要使用任何 Vue Plugins，首先要在 plugins (opens new window) 目錄底下新增.js檔，我們命名為vuex-persist.js:

// plugins/vuex-persist.js
import VuexPersistence from 'vuex-persist'

export default ({ store }) => {
  new VuexPersistence({
    storage: window.localStorage,

    // Optional，設定 state 在 LocalStorage 裡的 key。
    key: 'user-setting' 
  }).plugin(store);
}

# Nuxt 設定注意事項

要注意的是，根據 Nuxt 官方的 Store Plugins 指引 (opens new window)裡，新增 plugins 到 store 的方法如下:

// store/index.js
import myPlugin from 'myPlugin'

// 直接 export 一個 plugins 變數
export const plugins = [myPlugin] 
export const state = () =({
  // Your state ...
})
export const mutations = {
  // Your mutations ...
}

# Step 4: 完成！

Vuex-Persist 最基本的設定只要簡單 3 步就完成了，以下的事情它都會自動幫我們處理好:

  1. 在 state 被 mutate 的時候，Vuex-Persist 會自動把 state 存到 LocalStorage，而不用我們手動透過 localStorage.setItem()來儲存。
  2. 在 store 初始化時，自動從 LocalStorage 抓取 state，並覆蓋 state 的 initial values，不用我們手動透過 localStorage.getItem() 來取值。因此當 user 打開網頁時，便會自動載入上次的設定(如: darkmode、個人資料等)。

需要注意的是，這只是基本設定，Vuex-Persist 會把 store 裡 所有 state 都存到 LocalStorage。

# Vuex-Persist 進階設定 - reducer

隨著專案規模變大，Vuex 的 state 數量也變得可觀，當中並非所有 state 都需要存在 LocalStorage，因此可以透過 reducer 來設定只儲存哪些 state 到 LocalStorage，除了避免不必要的 bugs 以外，也可以節省 client 端的儲存空間。

舉例來說，我們的 store 設定如下:

// store/index.js
export const store = () => ({
    darkmode: false, // darkmode
    username: 'Isaac Kwok',
    cartItems: [] // 購物車內容
})
export const mutations = () => ({
    // mutations ...
})
export const actions = () => ({
    // actions ...
})

而假設我們只打算把 username 跟 darkmode 設定存起來，購物車內容只保留到網頁關掉，可以設定reducer:

// plugins/vuex-persist.js
import VuexPersistence from 'vuex-persist'

export default ({ store }) => {
  new VuexPersistence({
    storage: window.localStorage,
    reducer: (state) => ({ 
        darkmode: state.darkmode,
        username: state.username
    })

    // Optional，設定 state 在 LocalStorage 裡的 key。
    key: 'user-setting' 
  }).plugin(store);
}
